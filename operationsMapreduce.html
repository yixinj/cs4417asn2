Writing the code

 

You can log into the VM and use one of the editors (e.g., vi) or you can use the editor on your own computer and use sftp to upload the code files to the VM. 

 

 

Running the code

 

Assumptions:

 

á      You are using Python. 

á      You have written the code.   The mapper code is in a file called mapper.py and the reducer code is in reducer.py. 

á      You should sftp any test data files to the VM.

á      For simplicity, commands below are being executed from your home directory.

á      Your user name is cloudera

 

 

1.     When you log into the VM you are using the Linux filesystem.  You need to copy the code to Hadoop Distributed File System (HDFS) which is a distributed file system.  We recommend that you create a directory to place your input files.   This can be done using the following command:

 

hadoop fs -mkdir /user/cloudera/inputAssignment1

 

Please note that you can use file name that you want.  

 

 The command to copy to the directory is the following:

 
    hadoop fs -copyFromLocal a.txt /user/cloudera/inputAssignment1/a.txt
 

      Please note that you can copy multiple files to the same directory.

 

      The command

 

   hadoop fs -ls   /user/cloudera/inputAssignment1

 

       allows you to see the contents of the directory you created.

      

 

2.      To run the program use the following command:

 

 hadoop jar /usr/lib/hadoop-0.20-mapreduce/contrib/streaming/hadoop-streaming-2.6.0-mr1-cdh5.12.0.jar

-file mapper.py -mapper mapper.py

-file reducer.py -mapper reducer.py

 -input /user/cloudera/inputAssignment1

-output /user/cloudera/outputAssignment1

 

 

 

 

Please note this command is one line Ð do not hit enter until you have entered all the text.

 

3.     If you want to see the results use the following command

 

hadoop fs -cat /user/cloudera/outputAssignment1/*

 

4.     In Mapreduce each reduce task write its output to a file named part-r-nnnnn where nnnnn is a partition identifier associated with the reduce task.  You can merge files using the getmerge command.  An example of its use is below:

 

hadoop fs -getmerge /user/cloudera/Inverted/output/* outputFile

 

Please note that outputFile is stored in the same directory that you run that Hadoop command.

 

 

5.     Mapreduce assumes all input comes from sys.stdin. Essentially Mapreduce will pipe your data files.  For some applications you need to know the file name.  You should use the following:

 

os.environ[Òmap_input_fileÓ]

 

        To use this function you must include import os in your program.

 

Important Note: When you are running a Hadoop job, the output directory must not already exist.   If  you try and run the command with an existing directory, Hadoop refuses to run the job. This is considered a feature of Hadoop. ItÕs designed to stop you inadvertently deleting or overwriting data that is already in the cluster. If you specify a different directory which doesnÕt already exist then the job will begin. LetÕs say you want to use an existing output directory.  You then need delete it using this command:
